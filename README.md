# MuscleMax Pro AI 💪🤖

**MuscleMax Pro AI** は、Google Apps Script (GAS) と Google スプレッドシートで動作する、完全サーバーレスな筋トレ記録・管理Webアプリケーションです。
ジムでのスマホ操作に特化したUIと、Googleの生成AI「Gemini」によるパーソナライズされたメニュー作成機能を搭載しています。

*※ここにスクリーンショットを貼ってください*

## ✨ 特徴

* **📱 スマホ特化の高速UI**: ジムでの記録を邪魔しない、サクサク動くモダンなデザイン（PWAライク）。
* **🤖 AIメニュー作成**: Gemini API (Flash Lite) を利用し、あなたの身長・体重・ジムの設備・当日のコンディションに合わせたメニューを瞬時に提案。
* **📅 履歴＆カレンダー**: トレーニング履歴をスプレッドシートに自動保存。カレンダー形式で「いつジムに行ったか」が一目瞭然。
* **📈 成長分析**: 部位ごとのバランスや、種目ごとの重量・総負荷量（Volume）の推移をグラフ化。
* **🧮 プレート計算機**: バーベルの目標重量に対して、左右にどのプレートを付ければいいかを視覚的に表示。
* **🔄 ルーティン管理**: よく行うメニューセットを保存し、ワンタップで呼び出し可能。
* **🔒 プライベート＆セキュア**: データは全てあなた個人のGoogleスプレッドシートに保存されます。

## 🛠️ 技術スタック

* **Frontend**: HTML5, Tailwind CSS, Chart.js, FontAwesome
* **Backend**: Google Apps Script (GAS)
* **Database**: Google Spreadsheet
* **AI**: Google Gemini API (gemini-2.5-flash-lite)

## 🚀 セットアップ手順

このアプリは無料で構築できます（Googleアカウントが必要です）。

### 1. スプレッドシートの準備

1. [Googleスプレッドシート](https://sheets.google.com/)を新規作成します。
2. シート名を `シート1` から **`Data`** に変更します。
3. 1行目に以下のヘッダーを入力します（A1〜H1）：
   `日付` | `部位` | `種目` | `重量` | `回数` | `セット数` | `1RM` | `メモ`

### 2. スクリプトの導入

1. スプレッドシートのメニューから **「拡張機能」** > **「Apps Script」** を開きます。
2. **`コード.gs` (Code.gs)** に、配布されているサーバー側コードをコピペします。
3. **`index.html`** を新規作成し、配布されているHTMLコードをすべてコピペします。

### 3. Gemini APIキーの取得 (AI機能用)

1. [Google AI Studio](https://aistudio.google.com/) でAPIキーを取得します（無料枠でOK）。
2. アプリの初回起動時、または `index.html` 内の `USER_CONFIG` にキーを設定します。

### 4. デプロイ

1. GAS画面右上の **「デプロイ」** > **「新しいデプロイ」** をクリック。
2. 「種類の選択」で **「ウェブアプリ」** を選択。
3. 設定:
   * **説明**: 任意（例: v1.0）
   * **次のユーザーとして実行**: **「自分」**
   * **アクセスできるユーザー**: **「自分のみ」** (推奨) または 「全員（Googleアカウントを持つユーザー）」
4. **「デプロイ」** をクリックし、発行されたURLをスマホで開きます。
   * ※初回のみ「承認が必要です」という画面が出ます。許可してください。
   * ※AI機能（外部通信）を使うため、スクリプト内で一度 `authCheck` 関数を実行して権限承認を行う必要がある場合があります。

## 📱 使い方

1. 発行されたURLをスマホのブラウザ（Chrome/Safari）で開きます。
2. **「ホーム画面に追加」** することで、ネイティブアプリのように全画面で使用できます。
3. **設定**: 最初にAIメニュー作成画面などで、身長・体重・ジムの設備・APIキーを入力すると、ブラウザに保存され次回から自動入力されます。

## ⚠️ 注意事項

* **APIキーの管理**: `index.html` 内にAPIキーを直接書き込んでGitHub等で公開（Public）設定にしないよう注意してください。公開する場合は必ずキーを空欄にしてください。
